Constructor Inheritance In Java

In Java, constructor inheritance is the process of creating a subclass that inherits the constructor of its parent class. Constructor inheritance can also be seen from various perspectives depending upon it's order of execution in the main method. One thing that must be kept in mind that the order of inheritance of constructor also depend on the type of constructor being used in the class.

Cases of Constructor Inheritance

There are broadly five cases in which the order of inheritance of constructor can be understood. The detailed description is as follows...

  • Case 1: When there is a non-parameterized constructor in the super class but no constructor in the subclass then the order will be something like this,
    Constructor in SuperClass: SuperClass()
    Constructor in SubClass: No constructor used
    Order of execution: Superclass()

  • Case 2: When there is non-parameterized constructor in both the super class and subclass then the order will be something like this,
    Constructor in SuperClass: SuperClass()
    Constructor in SubClass: Subclass()
    Order of execution: Superclass() Subclass()

  • Case 3: When there is non-parameterized constructor in both the super class and subclass, and a parameterized in the super class then, the order will be something like this,
    Constructor in SuperClass: SuperClass() SuperClass(Arguments)
    Constructor in SubClass: SubClass()
    Order of execution: Superclass() SubClass()
    Note: The parameterized constructor becomes idle as no super keyword is mentioned in the subclass to access it.

  • Case 4: When there is a non-parameterized constructor in the sub class, and a parameterized in the super class then, the order will be something like this,
    Constructor in SuperClass: SuperClass(Arguments)
    Constructor in SubClass: SubClass()
    Order of execution: ERROR
    Note: There will be an error because no "super" keyword has been used in the sub class to access the elements of the superclass.

  • Case 5: When there is no non-parameterized constructor in both the super class and subclass, and parameterized in both the super class and sub class then, the order will be something like this,
    Constructor in SuperClass: SuperClass(Arguments)
    Constructor in SubClass: SubClass(Arguments)
    Order of execution: ERROR
    Note: There will be an error until and unless the "super" keyword is used in the sub class because it will not be able to access the super class data members and methods.
